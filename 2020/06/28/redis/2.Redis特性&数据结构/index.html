Redis特性&数据结构

发表于 2020-06-28 | 分类于 Redis

1. Redis特性

1.速度快

  • Redis所有数据存放在内存中
  • Redis使用C语言编写,更偏向底层
  • Redis使用了单线程架构，预防了多线程可能产生的竞争问题

2.基于键值对数据结构

  • Redis基于键值对数据结构
  • 支持5种常用数据结构:字符串,哈希,列表,集合,有序集合
  • 支持位图（Bitmaps）和 HyperLogLog 两种数据结构

3.丰富功能

  • 提供键过期功能,可以用来实现缓存
  • 提供发布订阅功能,可以用来实现消息系统
  • 支持Lua脚本,可以通过Lua生成新的Redis命令
  • 提供简单事务功能,一定程度保证事务特性
  • 提供流水线(Pipiline)功能,可以批量执行Redis命令,减少网络开销

4.简单稳定

  • Redis源代码很少,早期只有2万行,3.0版本支持集群特性到5万代码

2. 数据结构和内部编码

Redis 常用的数据结构包括：string（字符串）、list（列表）、hash（哈希）、set（集合）、zset（有序集合）。

其中每种数据结构都有自己底层的内部编码实现，并且是多种实现，Redis根据使用场景选择不同的内部编码。

string：

  • raw
  • int
  • embstr

hash：

  • hashtable
  • ziplist

list：

  • linkedlist
  • ziplist

set：

  • hashtable
  • intset

zset：

  • skiplist
  • ziplist
