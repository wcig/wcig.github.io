Redis常见问题

发表于 2020-06-28 | 分类于 Redis

1. 缓存穿透、缓存击穿、缓存雪崩

缓存穿透

  • 问题描述：需要查询的在缓存中没有，即穿透了 redis 缓存去数据库中查找。

  • 解决办法：（1）如果为单条或少量的数据穿透，可以将这些数据以空数据的形式缓存在 redis 中，当数据库数据有变化再更新 redis 数据；（2）如果为大量的数据穿透可以使用布隆过滤器。（低频的缓存穿透可以不解决，高频的需要解决）

缓存击穿

  • 问题描述：原先在 redis 中缓存有热门数据，当该热门数据失效时将穿过缓存区数据库中查找。
  • 解决办法：问题不是特别严重的可以不解决，如果需要解决可以使用分布式锁来解决。

缓存雪崩

  • 问题描述：即缓存崩溃。可能出现的情况包括：大量的热点数据同时失效；redis 奔溃。
  • 解决办法：如果是大量热点数据同时失效则可以将数据设置为随机不同的失效时间；如果是 redis 奔溃则可以替换为 redis 集群来提供服务。

区别：

  • 缓存击穿、缓存雪崩是缓存穿透的2种特殊表现形式。
