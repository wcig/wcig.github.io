Redis阻塞

发表于 2020-06-28 | 分类于 Redis

Redis是单线程架构，所有读写操作都是在一个线程中执行，所以在高并发场景下阻塞将会导致很大的问题。

1.发现阻塞

如何发现阻塞，一方面是通过Redis监控工具进行监控，一方面是通过收集应用连接Redis的错误日志发现。

2.内在原因

Redis阻塞可能是由于Redis使用导致的，可能存在的原因包括：

  • 错误数据结构或不合理的数据（发现慢查询、发现大对象）
  • Redis使用CPU已经跑满（监控Redis进程）
  • 持久化时导致的阻塞（通过Redis内部命令）

3.外在原因

  • CPU竞争
  • 内存交换
  • 网络问题
