3.程序结构

  • go 包结构通过 package 组织
  • 只有 package 为 main 的包可以包含 main 函数
  • 一个可执行程序只有一个 main 包
  • import 关键字引入其他包
  • const 关键字定义常量
  • var 关键字定义全局变量
  • type 关键字进行 struct 或 interface 的声明
  • func 关键字进行函数声明

4.package 下划线引入：
当导入一个包时，该包下的文件里所有init()函数都会被执行，然而有些时候我们并不需要把整个包都导入进来，仅仅是是希望它执行init()函数而已。这个时候就可以使用 import _ 引用该包。即使用【import _ 包路径】只是引用该包，仅仅是为了调用init()函数，所以无法通过包名来调用包中的其他函数。

使用规则：

  • 一般使用常规引入
  • 当引入的多个包 package 一样时可使用 package 别名
  • 一般不建议使用 . 省略 package 引入

5.Go 包初始化

Go 初始化顺序:(1)引入的包;(2)当前包的变量常量;(3)当前包的init函数;(4)main函数.

6. Go 可见性

Go 语言使用大小写来决定常量、变量、类型、接口、结构或函数是否可以被外部包所访问。
eg：当函数名首字母为小写，即为 private；当首字母为大写，即为 public。
