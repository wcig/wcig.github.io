Go切片

发表于 2020-07-29 | 分类于 Golang

1. 概念

由于数组长度是固定的，在一些场景不太适用，go 提供了切片 slice，它长度不是固定的，可以动态扩容和缩容。

slice 特征：

  • 长度不固定，可扩容和缩容
  • 为引用类型
  • 本身不是数组，而是指向底层数组
  • 内置 len() 函数获取 slice 元素个数，cap() 获取 slice 容量
  • 多个 slice 指向相同底层数组，其中一个值的改变会影响全部 slice

3. slice 初始化

  • 切片 slice 可以直接初始化，也可以通过数组初始化
  • s := arr[:]，切片 s 是 arr 数组的引用
  • s := arr[startIndex:]，表示将 arr 数组的 startIndex 到 arr 的最后一个元素作为切片（等价于：arr[startIndex:len(arr)-1]）
  • s := arr[:endIndex]，表示将 arr 数组的第一个元素到 endIndex-1 的元素作为切片（等价于：arr[0:endIndex]）
  • s := arr[startIndex:endIndex]，表示将 arr 数组的 startIndex 到 endIndex-1 的元素作为切片

4. len()、cap()

  • 通过数组来初始化切片，切片长度 len() 为切片元素个数
  • s := arr[:]，切片 s 容量 cap() 等于数组长度
  • s := arr[startIndex:]，切片 s 容量为从 startIndex 到数组最后一元素大小
  • s := arr[:endIndex]，切片 s 容量为数组长度
  • s := arr[startIndex:endIndex]，切片 s 容量为从 startIndex 到数组最后一元素大小

5. Reclice 从一个切片初始化另一个切片

  • reslice 时索引以被 slice 切片为基准
  • reslice 后2个 slice 地址是不同的
  • 多个 slice 指向相同数组，或一个 slice 指向另一个 slice 时，其中一个修改值，所有都会被修改
  • relice 时，创建的 slice 切片不能超过被 slice 的容量 cap
  • 索引越界不会导致数组的重新分配，而是报错

6. append()

  • append() 是在切片尾部添加元素
  • 添加元素后切片长度大于容量时，容量扩容为原来容量的1倍，一旦元素个数超过 1000，容量的增长因子会设为 1.25，也就是会每次增加 25%的容量。随着语言的演化，这种增长算法可能会有所改变
  • append() 添加元素扩容后，将重新分配数组并拷贝数据，此时地址将发生改变
  • 空切片也可使用 append() 添加元素

7. 删除元素

go 中切片没有现有的删除元素的方法，只能自己实现。实现方式有两种，一种是通过切片位置元素的移动来实现保持原有顺序的删除，一种是直接交换把需要删除元素交换最后一个元素，但这样不能保持原有顺序。

10. 使用 range 遍历切片

  • 当切片为数组时，通过 range 遍历，value 对应元素值的副本，而不是指向元素的引用，所以此时不能通过 value 设置值来修改数组，而应该通过索引来进行修改
  • 当切片为指针数组时，通过 range 遍历，value 为数组的各个指针元素，可以直接通过 value 进行修改，也可以通过索引
