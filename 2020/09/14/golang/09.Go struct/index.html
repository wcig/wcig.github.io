Go struct

发表于 2020-09-14 | 分类于 Golang

1. struct特征

  • Go 中 struct 与 C 中的 struct 类似，Go 中没有 class
  • 使用 type struct{} 定义结构，名称遵循可见性原则
  • 支持指向自身的指针类型成员
  • 支持匿名字段，可用作成员或定义成员变量
  • 匿名字段可以用于 map 的值
  • 可以使用字面值对结构进行初始化
  • 允许直接通过指针来读写结构成员变量
  • 相同类型的成员可进行直接拷贝赋值
  • 支持 == 与 != 比较运算符，但不支持 < 或 >
  • 支持匿名字段，本质上是定义了某个类型名为名称的字段
  • 嵌入结构作为匿名字段看起来像继承，但不是继承
  • 可以使用匿名字段指针

2. struct定义、初始化

  • 使用 type struct{} 定义结构，名称遵循可见性原则
  • 可以直接初始化结构体值，也可以初始化结构体指针
  • 结构体初始化时未赋值，所有成员变量将默认初始化为零值

3. struct成员变量

  • 结构体成员变量遵循可见性原则
  • 结构体的成员变量可以通过结构体对象和结构体指针来访问和修改

4. 传递struct值类型、引用类型

  • struct值类型在函数中作为参数传递不会被修改，引用类型可以被修改
  • 造成struct值类型和引用类型是否可被修改的区别，在于函数传参时都是拷贝传入的参数，拷贝后的引用类型与原值指向同一块内存区域。

5. struct比较

  • struct 可以比较（==、!=），但不可以排序（>、>=、<、<=）

  • struct 是只有相同类型才可以比较，不同类型不能比较

  • stuct 值类型比较，当所有值相等即为 true；struct 引用类型比较需要指向同一块内存区域才为 true

  • struct 只有可以比较时才可以作为 map 的 key

8. struct强转

json字符串：驼峰-下划线

  • 当两个struct所有成员变量都一样只有tag不一样时，可以直接通过值类型强转
  • 如果2个struct有成员变量不一致强转将编译不通过
  • struct强转可以用于json tag驼峰和下划线之间转换

9. struct浅拷贝与深拷贝

  • 当struct的成员变量均为基础类型时，浅拷贝与深拷贝一致。
  • 当struct存在成员变量为引用类型时，此时浅拷贝与深拷贝存在区别，区别的地方在于浅拷贝前后2个值的引用类型成员变量依然指向同一块内存区域，而深拷贝是各自单独的。
  • 一些github的深拷贝的库实际测试并不能真正实现深拷贝，所以使用前建议多做测试，一般不复杂的struct建议单独写一个深拷贝的函数。
