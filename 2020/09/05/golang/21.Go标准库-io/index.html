3.1 Reader, Writer接口

Reader 接口为包含基本读取方法的接口, 各种文件的读取都实现了这个接口.

Read 方法读取 len(p) 个字节到 p 中, 返回读取到的字节数以及错误. 错误可能是错误, nil 和
EOF. 如果需要实现该接口, 建议返回的 n 和 err 是互斥的.

Writer 接口为包含基础写入方法的接口.

Write 方法从 p 中读取 len(p) 个字节进行写入, 返回写入成功的字节数以及错误.

3.2. Closer接口

Closer 接口包装了基本的关闭方法.

文件, 数据库连接等需要手动关闭资源都实现了该接口. 编程中一般配合 defer 使用.

3.3 Seeker接口

Seeker 接口包装了基本的偏移方法.

Seek 方法偏移指定偏移量, 用于下一次的读取或写入. offset 指定偏移量, whence 指定起始位置, 返回新的相对于文件起始的偏移量和错误:

  • os.SeekStart: 相对于文件起始
  • os.SeekCurrent: 相对于文件当前位置
  • os.SeekStop: 相对于文件结尾位置

3.5 ReaderForm, WriterTo接口

ReaderFrom 和 WriterTo 接口都是只包装一个读和写方法的接口.

ReaderForm 接口的 ReadForm 方法, 从 r 中读取数据直到遇到 EOF 或错误, 返回读取的字节数和错误. 除 io.EOF 外, 读取过程中遇到的任何错误都将返回.

WriterTo 接口的 WriteTo 方法, 将写入数据到 w, 直到没有数据写入或发生错误. 返回写入的字节数和错误, 写入过程出现的任何错误都将返回.

3.6 ReaderAt, WriterAt接口

ReaderAt 是一个封装的基本接口 ReadAt 方法。

  • ReadAt 从输入源偏移量 off 处, 读取 lens(p) 个字节到 p 中. 它返回的字节数读 (0 <= n <= len(p))) 和所遇到的任何错误。
  • 当 ReadAt 返回 n<len(p) 时，它返回一个非零错误解释为什么不返回多个字节。 在这方面， ReadAt 比Read更严格。
  • 即使 ReadAt 返回 n<len(p) 时，它可以在呼叫过程中使用所有的p作为暂存空间的。 如果一些数据可用但不到 len(p) 个字节， ReadAt 块将直到所有的数据是可用的或发生错误。 在这方面 ReadAt 与 Read 是不同的。
  • 如果 n=len(p)， ReadAt 可能返回 err=EOF 或 err=nil。
  • 如果 ReadAt 为从输入源带偏移量的读取， ReadAt 将不影响偏移量同时也不被偏移量影响。
  • 可以对同一个输入源并行 ReadAt 调用。

WriterAt 是一个封装的基本接口 WriteAt 方法。

  • WriteAt 从 p 中将 len(p) 个字节写入到偏移量 off 处的基本数据流中。它返回从 p 中被写入的字节数 n（0 <= n <= len(p)）以及任何遇到的引起写入提前停止的错误。若 WriteAt 返回的 n < len(p)，它就必须返回一个 非nil 的错误。
  • 若 WriteAt 携带一个偏移量写入到目标中，WriteAt 应当既不影响偏移量也不被它所影响。
  • 若被写区域没有重叠，可对相同的目标并行执行 WriteAt 调用。

3.7 其他接口

ByteReader, ByteWriter 为读取或写入一个字节的接口。

ByteScanner 为包装了 ByteReader 接口和 UnreadByte 方法的接口。

调用 UnreadByte 方法后，调用 ReadByte 方法读取到的字节将于前一次调用一样。

RuneReader 为包装了读取一个 UTF-8 unicode 字符方法的接口。

RuneScanner 接口为包装了 RuneReader 接口 和 UnreadRune 方法的接口。其中 UnreadRune 方法与 ByteScanner 接口的 UnreadByte 方法功能类似。

StringWriter 为包装了一个写入字符串方法的接口。

5. 函数

5.1 Copy, CopyN, CopyBuffer

Copy() 方法从 src 拷贝数据到 dst 中, 返回拷贝成功的字节数和拷贝过程中出现的第一个错误.

  • 成功的拷贝返回的 err == nil, 而不是 EOF. 因为 Copy 的定义就是从 src 读取数据直到 EOF, 当读取遇到 EOF 将不会认为是个错误.
  • 如果 src 实现了 WriterTo 接口, 拷贝过程中将直接调用 src.WriteTo(dst). 如果 dst 实现了 ReaderFrom 接口, 则拷贝过程中将直接调用 dst.ReadForm(src).

CopyN() 方法从 src 拷贝 n 个字节到 dst 中, 返回拷贝成功的字节数和期间出现的错误.

  • 当返回 written == n 时, err == nil.
  • 如果 dst 实现了 ReaderForm 接口, 拷贝时将使用 ReadForm 方法.

CopyBuffer() 方法与 Copy() 方法类似, 唯一的区别是带缓冲的拷贝. 如果 buf 为 nil, 则默认分配一个字节的缓冲, 如果 len(buf) 将会 panic.

5.2 ReadAtLeast, ReadFull

ReadAtLeast 方法从 r 中读取数据到 buf 中，直到读到最少 min 个字节。函数返回读取到的字节数和错误。

  • 只有当没有字节可以读取时错误是 EOF
  • 如果一个 EOF 发生在读取小于 min 字节的数据，将返回错误 ErrUnexpectedEOF
  • 如果 min 大于 buf 的长度，将返回错误 ErrShortBuffer
  • 当且仅当 err == nil 时，n >= min
  • 如果已经读取了 min 字节数据时 r 返回错误，此时错误将会被忽略

ReadFull 方法从 r 读取 len(buf) 字节数据到 buf 中，函数返回读取的字节数和错误。

  • 如果错误为 EOF，则只有没有字节数据读取的情况
  • 如果 EOF 发生在读取到部分但不是所有字节数据时，返回错 ErrUnexpectedEOF
  • 返回值 n == len(buf) 只在 err == nil 情况发生
  • 如果已经读取了 len(buf) 字节数据时 r 返回错误，此时错误将会被忽略

ReadAtLeast 与 ReadFull 的唯一区别就是 ReadFull 是将 buf 读满，而 ReadAtLeast 是最少读取 min 字节数。

5.3 WriteString

WriteString 方法是将字符串 s 写入到 w 中，返回写入成功的字节数和错误。

当 w 实现了 StringWriter 接口，将直接调用 WriteString 方法，否则调用 w.Write([]byte(s))。

5.5 TeeReader

TeeReader 返回一个 Reader，它将从 r 中读到的数据写入 w 中。所有经由它处理的从 r 的读取都匹配于对应的对 w 的写入。它没有内部缓存，即写入必须在读取完成前完成。任何在写入时遇到的错误都将作为读取错误返回。
