3.1 bufio.Reader

bufio.Reader 结构包装了一个 io.Reader 对象来提供缓存读功能，同时自身内部实现了 io.Reader 接口。

类型定义：

实例化：

NewReader() 函数接收一个 reader 参数，以默认换成大小转换成 bufio.Reader。

NewReaderSize() 函数接收一个 reader 和一个 size 大小，按照指定 size 大小转换成 bufio.Reader。

方法：

ReadSlice 方法：从输入中读取直到遇到第一个界定符（delim），返回一个包含界定符指向缓存中的字节切片，下次调用读操作这些字节会失效。

  • 如果在找到界定符之前出现错误，将会返回 buffer 中的数据和出现的错误（经常是 io.EOF）
  • 如果在找到界定符之前 buffer 就已经满了，将会返回错误 ErrBufferFull
  • 如果返回的 line 没有以界定符结尾，则 err != nil
  • 因为返回的字节切片是直接指向 Reader 的 buffer 的，而不是拷贝出一份，所以 ReadSlice 读取的数据会被下一次 I/O 操作重写。建议使用 ReadBytes 或 ReadString代替。

示例：

ReadBytes 方法的参数和返回值与 ReadSlice 一致。也是从输入读取直到遇到界定符（delim）为止，返回的数据同样也包括界定符。

  • 如果在找到界定符之前出现错误，将返回错误出现前读取的数据以及该错误（经常是 io.EOF）
  • 如果返回的 line 没有以界定符结尾，则 err != nil
  • 方法返回的字节切片数据没有指向 buffer，所以不会像 ReadSlice 那样下次读取会影响上次读取到的数据

示例：

ReadString 方法与 ReadBytes 类似，区别在于返回的不是字节切片而是字符串。

  • 如果在找到界定符之前出现错误，将返回错误出现前读取的数据以及该错误（经常是 io.EOF）
  • 如果返回的 line 没有以界定符结尾，则 err != nil
  • 方法返回的字节切片数据没有指向 buffer，所以不会像 ReadSlice 那样下次读取会影响上次读取到的数据

示例：

ReadLine 是一个低水平的粗糙的行数据读取方法。大多数调用者应使用ReadBytes(‘\n’)或ReadString(‘\n’)代替，或者使用Scanner。

  • ReadLine 尝试返回一行数据，不包括行尾标志的字节。如果行太长超过了缓冲，返回值 isPrefix 会被设为 true，并返回行的前面一部分。该行剩下的部分将在之后的调用中返回。返回值 isPrefix 会在返回该行最后一个片段时才设为false。返回切片是缓冲的子切片，只在下一次读取操作之前有效。ReadLine 要么返回一个非 nil 的 line，要么返回一个非 nil 的 err，两个返回值至少一个非 nil。
  • 返回的文本不包含行尾的标志字节（”\r\n”或”\n”）。如果输入流结束时没有行尾标志字节，方法不会出错，也不会指出这一情况。在调用 ReadLine 之后调用 UnreadByte 会总是吐出最后一个读取的字节（很可能是该行的行尾标志字节），即使该字节不是 ReadLine 返回值的一部分。

3.2 Writer

bufio.Writer 包装了一个 io.Writer 对象的提供缓存写功能的结构，同时也实现了 io.Writer 接口。

类型定义：

如果在向一个Writer类型值写入时遇到了错误，该对象将不再接受任何数据，且所有写操作都会返回该错误。在说有数据都写入后，调用者有义务调用Flush方法以保证所有的数据都交给了下层的io.Writer。

3.3 bufio.Scanner

bufio.Scanner 类型提供了方便的读取数据接口，如从换行符分隔的文本读取每一行。

类型定义：

函数：

SplitFunc 类型表示对输入做词法分析的分割函数。

  • 参数 data 表示待处理的字节切片
  • 参数 atEOF 表示是否 Reader 接口不能提供更多数据（是否到EOF了）
  • 返回值 advance 表示解析位置前进的字节数，token 表示下一个结果数据，err 表示错误
  • 如果 data 没有一个完整的 token，则返回 (0, nil. nil)

例如：hello world.\tok\t666 通过 \t 分词后会得到 3 个 token，分别是 “hello world.”、“ok” 和 “666”，而上面返回的是第一个 token 即 “hello world.”。

方法：

Scan 方法：获取当前位置的 token（token 可以通过 Bytes 或 Text 方法获取），并让 Scanner 的扫描位置移动到下一个 token（有点类似于 Iterator 的 next）。

  • 当扫描因到达 Reader 末尾或遇到错误停止时，本方法会返回 false。
  • Scan 方法返回 false，此时 Err 方法会返回扫描时出现的错误，如果是 io.EOF，Err 方法将返回 nil。
