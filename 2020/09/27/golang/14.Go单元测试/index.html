1. 单元测试

go 语言通过 testing 包可以实现自动化的测试。一般规则就是：

  • 单元测试文件需以 _test.go 后缀结尾
  • 文件的函数格式应形如：func TestXxx(*testing.T)
  • 测试文件跟要测试的代码文件在同一个目录下
  • 使用 go test 命令即可运行测试样例

3个计时方法：

  • b.StartTimer：开始计时测试。在基准测试开始之前会自动调用此函数，但也可以用于在调用StopTimer之后恢复计时。

  • b.StopTimer：停止计时测试。这可用于在执行您不想测量的复杂初始化时暂停计时器。

  • b.ResetTimer：将经过的基准时间和内存分配计数器归零，并删除用户报告的指标。它不影响计时器是否正在运行。

9. 日志输出

testing.T 类型提供了一些日志输出的工具方法，在介绍工具方法之前介绍3个方法：

通过上面示例我们可以得出以下结论：

  • t.Fail()：将测试程序标记为失败并继续执行后面的代码
  • t.FailNow()：将测试程序标记为失败并结束
  • t.Failed()：反馈测试程序是否失败
  • t.SkipNow()：跳过后面代码结束程序
  • t.Skipped()：反馈程序是否跳过

日志输出示例：

通过以上示例可以得出日志输出方法为：

  • t.Log()：正常输出日志，只有在测试失败和 -test.v 设置时打印。
  • t.Error()：等价于 t.Log() + t.Fail()。
  • t.Fatal()：等价于 t.Log() + t.FailNow()。
  • t.Skip()：等价于 t.Log() + t.SkipNow()。
